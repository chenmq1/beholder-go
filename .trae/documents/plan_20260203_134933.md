## 问题分析

通过分析代码，我发现了问题的根本原因：**链 ID 映射错误**。

- 在 `burnpair/code_get_service.go` 文件中，调用 `GetCode` 方法时使用的链 ID 是 2，注释说明 "burnPair only occurs on chainId 2 (bsc)"
- 但在 `get_code_service.go` 文件中，`getChainName` 函数的链 ID 映射中，BSC 对应的链 ID 是 56，不是 2
- 当链 ID 为 2 时，`getChainName` 函数会返回默认值 "ethereum"，导致使用以太坊节点查询 BSC 地址的代码

## 解决方案

按照用户要求，修改链 ID 映射，使用新的链 ID 映射：
- BSC: 2
- Polygon: 3
- Arbitrum: 4
- Base: 5
- Optimism: 6

## 具体修改

1. **文件路径**: `internal/service/getcode/get_code_service.go`

2. **修改内容**: 
   - 修改 `getChainName` 函数，更新链 ID 映射
   - 完全替换原来的 chainMap，使用新的链 ID 映射

3. **修改代码**: 
   ```go
   // getChainName 根据链 ID 获取链名称
   func (s *GetCodeService) getChainName(chainId int) string {
       // 更新链 ID 映射
       chainMap := map[int]string{
           1:    "ethereum",
           2:    "bsc",      // BSC 对应链 ID 2
           3:    "polygon",   // Polygon 对应链 ID 3
           4:    "arbitrum",  // Arbitrum 对应链 ID 4
           5:    "base",      // Base 对应链 ID 5
           6:    "optimism",  // Optimism 对应链 ID 6
       }

       if name, ok := chainMap[chainId]; ok {
           return name
       }

       return "ethereum" // 默认以太坊
   }
   ```

## 预期结果

修改后，当使用链 ID 2 调用 `GetCode` 方法时，`getChainName` 函数会返回 "bsc"，代码会使用 BSC 节点获取合约代码，避免所有地址的 binary code 都是 "0x" 的问题。
