# 修复getDecompiled方法中的URL硬编码问题

## 问题分析

当前 `getDecompiled` 方法存在以下问题：

1. **URL 硬编码**：使用固定的测试 URL `https://app.dedaub.com/binance/address/0xd827820a13ff041c43b0b9068aa4d3590b35826f/decompiled`
2. **链 ID 处理不当**：硬编码使用 "binance" 前缀，没有根据 `chainId` 选择不同的链前缀
3. **未使用配置信息**：没有利用 `s.chains` 中的 `GetDedaubDecompileURL` 方法来构建 URL

## 解决方案

### 修改 `getDecompiled` 方法

1. **获取标准链 ID**：使用 `s.getStandardChainId(chainId)` 获取标准链 ID
2. **获取反编译 URL 模板**：使用 `s.chains.GetDedaubDecompileURL(standardChainId)` 获取反编译 URL 模板
3. **构建动态 URL**：使用 `fmt.Sprintf` 根据地址格式化 URL

### 具体修改


## 预期结果

- **动态 URL 构建**：根据传入的地址和链 ID 构建正确的反编译 URL
