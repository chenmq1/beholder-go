## 问题分析

在 Java 版本的 `ContractCode` 实体类中，二进制代码的字段名是 `binaryCode`，对应的数据库列名也是 `binaryCode`。

在 Go 版本的 `ContractCode` 模型中，二进制代码的字段名是 `Binary`，GORM 会默认将其映射为数据库列名 `binary`。

当 GORM 尝试插入或更新 `binary` 列时，MySQL 会报错 "Unknown column 'binary' in 'field list'"，因为数据库中实际的列名是 `binaryCode`。

## 解决方案

修改 Go 版本的 `ContractCode` 模型，将 `Binary` 字段的 GORM 标签设置为正确的列名 `binaryCode`。

## 具体修改

1. **文件路径**: `internal/model/contract_code.go`

2. **修改内容**: 将 `Binary` 字段的定义从:
   ```go
   Binary                   string    `json:"binary"`
   ```
   修改为:
   ```go
   Binary                   string    `json:"binary" gorm:"column:binary_code"`
   ```

3. **验证**: 运行 `go build ./...` 命令验证编译是否通过

## 预期结果

修改后，GORM 会将 `Binary` 字段正确映射到数据库的 `binary_code` 列，避免 "Unknown column 'binary' in 'field list'" 错误。
